**改动 3**：reducer 实现 16 条 transition

按 20260513-4 §3.3 transition 表逐条对照实现。关键点：

- `toggle-mode` 同 mode 关闭时透传 `trigger='button'`（除非 action 显式带其它 trigger）。
- `close` 按 §3.4 子表分级：
  - `runtime` / `context` / `progress` + trigger='esc' → `isOpen=false, pinnedMode=null`
  - `artifact` + trigger='esc' → 若 `lastNonBlockingMode === 'context'` 则切到 context 且 `pinnedMode=null`；否则 `isOpen=false, pinnedMode=null`
  - `memory` + trigger='esc' → 切到 context，`pinnedMode=null`
  - `approval` + 任意 trigger → 不变（return state）
  - 任意非 approval + trigger='button' 或缺省 → `isOpen=false, pinnedMode=null`
- `request-approval`：保存 `lastNonBlockingMode = state.mode === 'approval' ? state.lastNonBlockingMode : state.mode`（防止 approval 嵌套覆盖恢复点）。
- `resolve-approval`：仅在 `state.mode === 'approval'` 时生效，否则 return state（T11）。
- `progress-event` / `artifact-draft-detected`：先按 transition 表第 6 / 7 / 8 条判断是否切 mode，再无条件写对应 ref（即使不切 mode，ref 也要更新）。
- `set-*-ref` 三组：只写 ref，绝不动 mode / isOpen / pinnedMode（T15 严格断言）。

**改动 4**：加 `assertInvariants` 开发期断言

```ts
function assertInvariants(state: RightRailState): void {
  if (!import.meta.env.DEV) return
  // I1
  const hasApprovalRef = state.approvalRef !== null
  const isApprovalMode = state.mode === 'approval'
  if (hasApprovalRef !== isApprovalMode) {
    throw new Error('[RightRail invariant I1] mode=approval iff approvalRef!=null')
  }
  // I2
  if (state.lastNonBlockingMode === 'approval') {
    throw new Error('[RightRail invariant I2] lastNonBlockingMode must not be approval')
  }
  // I3
  if (state.pinnedMode !== null && !['artifact', 'memory', 'runtime'].includes(state.pinnedMode)) {
    throw new Error('[RightRail invariant I3] pinnedMode must be null or user-inspect mode')
  }
  // I4
  if (!state.isOpen && state.mode !== state.lastNonBlockingMode) {
    throw new Error('[RightRail invariant I4] closed state must align mode with lastNonBlockingMode')
  }
}
```

在 reducer 的每个 case `return` 前包一层：

```ts
function withInvariants(state: RightRailState): RightRailState {
  assertInvariants(state)
  return state
}
// 所有 return 改成 return withInvariants(nextState)
```

注意 I4 可能与现有 `close` 行为冲突——当前 reducer close 时不调整 `lastNonBlockingMode`，可能与 `mode` 不一致。Phase 2 实现时需要在 close 路径上同步 `lastNonBlockingMode = nextMode`，确保不变量成立。

**改动 2**：派生 `openDocument` 显示对象

新增一个 useMemo，从 `rightRailState.artifactRef` + `currentArtifacts` + `currentSessionKey` + 消息附件信息派生出渲染分支需要的"显示对象"。这个对象**不是状态**，是派生值：

```ts
const openDocument = useMemo<OpenDocument | null>(() => {
  const ref = rightRailState.artifactRef
  if (!ref || rightRailState.mode !== 'artifact') return null
  if (ref.kind === 'artifact') {
    // 从 currentArtifacts 找
    const found = currentArtifacts.find(
      (a) => a.messageId === ref.messageId && a.artifactIndex === ref.indexInMessage,
    )
    if (!found) return null
    return {
      kind: 'artifact',
      key: ref.key,
      messageId: ref.messageId,
      artifactIndex: ref.indexInMessage,
      sessionKey: ref.sessionKey ?? '',
      artifact: found.artifact,
    }
  }
  // attachment 路径：从消息列表里反查 attachment
  // 当前 HomePageLayout 没有直接持有 messages，attachment 只在打开时由 ConversationArea 通过 onOpenAttachment 回调传入。
  // 因此 attachment 派生需要保留 attachment 对象的"快照"——把 attachment 对象直接挂到 RightRailArtifactRef 上不优雅但务实。
  // 折中：扩 RightRailArtifactRef 加可选 attachmentSnapshot 字段，仅 kind='attachment' 时填。
  return null
}, [rightRailState.artifactRef, rightRailState.mode, currentArtifacts])
```

**关于 attachment 派生**：当前 `openDocument.kind === 'attachment'` 路径的 attachment 对象来自 `ConversationArea` 的 `onOpenAttachment` 回调（`HomePageLayout.tsx:647-665`）。attachment 不在 `currentArtifacts` 镜像里。最务实的方案是扩 `RightRailArtifactRef`：

```ts
export interface RightRailArtifactRef {
  kind: 'attachment' | 'artifact'
  key: string
  sessionKey?: string | null
  messageId: string
  indexInMessage: number
  attachmentSnapshot?: ChatAttachment  // 仅 kind='attachment' 时填
}
```

`attachmentSnapshot` 是数据快照，不是状态——快照内容固定不变，写入时拷贝。Artifact mode 规范在 Phase 4 / 5 落地时会用更优雅的派生方式替换，本阶段先用快照。

**改动 4**：重构 draft auto-open useEffect（按 20260513-4 §4.2）

`HomePageLayout.tsx:690-702` 当前实现包含三个判断：`activeView !== 'chat'`、`openDocument`、`rightRailState.pinnedMode`。重构为：

```ts
useEffect(() => {
  if (activeView !== 'chat') return
  const nextDraft = currentArtifacts.find(({ artifact }) => {
    if (artifact.status !== 'draft') return false
    const draftKey = artifact.stepId ?? artifact.artifactId
    if (seenDraftArtifactKeysRef.current.has(draftKey)) return false
    seenDraftArtifactKeysRef.current.add(draftKey)
    return true
  })
  if (!nextDraft) return

  dispatchRightRail({
    type: 'artifact-draft-detected',
    ref: {
      kind: 'artifact',
      key: `${nextDraft.messageId}:artifact:${nextDraft.artifactIndex}`,
      sessionKey: selectedSessionKey ?? currentSessionKey,
      messageId: nextDraft.messageId,
      indexInMessage: nextDraft.artifactIndex,
    },
  })
}, [activeView, currentArtifacts, currentSessionKey, selectedSessionKey])
```

注意：

- **不要**在 useEffect 中读 `rightRailState.pinnedMode` 或 `rightRailState.mode` 判断——reducer 内 transition 第 7 条会自动处理 pinned 情况下"只写 ref 不切 mode"。
- `seenDraftArtifactKeysRef` 保留作为去重，避免反复 dispatch 同一 draft。

**改动 5**：artifact mode mirror useEffect

`HomePageLayout.tsx:620-645` 当前那段"`currentArtifacts` 变化时刷新 `openDocument` artifact 引用"的 useEffect 改为：

```ts
useEffect(() => {
  if (rightRailState.artifactRef?.kind !== 'artifact') return
  const ref = rightRailState.artifactRef
  const latestLocation = findLatestArtifactLocation(
    currentArtifacts,
    /* 用 ref 反查的旧 artifact——这里需要从 currentArtifacts 拿当前对象 */
    currentArtifacts.find(
      (a) => a.messageId === ref.messageId && a.artifactIndex === ref.indexInMessage,
    )?.artifact,
  )
  if (!latestLocation) return
  if (
    latestLocation.messageId === ref.messageId
    && latestLocation.artifactIndex === ref.indexInMessage
  ) return
  dispatchRightRail({
    type: 'set-artifact-ref',
    ref: {
      ...ref,
      key: `${latestLocation.messageId}:artifact:${latestLocation.artifactIndex}`,
      messageId: latestLocation.messageId,
      indexInMessage: latestLocation.artifactIndex,
    },
  })
}, [currentArtifacts, rightRailState.artifactRef])
```

如果 `findLatestArtifactLocation` 签名不支持上述用法，调整传参方式（参考 `findLatestArtifactLocation` 实现）。

## 0. 适用范围

本规范只描述 `SettingsDrawer.tsx`（右侧设置抽屉）的**视觉与交互重设计**。当前阶段**只产出规范、不动代码**；后续落地需要单独的执行指令。

> CLAUDE.md §2.1 把 "frontend 不主动维护" 写成长期红线，但本次属于 kira 显式发起的设计任务，不属于"工具自发改进 UI"。

## 1. 背景与现状问题

当前 `SettingsDrawer.tsx`（935 行）的主要问题：

| 维度 | 现状 | 问题 |
|------|------|------|
| 层级 | `isContextPanelOpen` + `isModelPanelOpen` 两个全屏覆盖式二级面板 | 用户从一级面板进入二级要"跳页"，关闭后状态丢失感强，违反"无二级页面" |
| 容器 | 每个分区都套 `railCardClassName`（`rounded-2xl border bg-surface shadow-...`），内部又叠 `rounded-xl border bg-surface-alt`，再叠 chip `rounded-xl border` | 三层"卡片包卡片"是典型气泡式堆叠，与极简工程风冲突 |
| 图标 | 引入 `Bot / Brain / Cpu / Database / FileText / SlidersHorizontal / Wrench / Pencil / Plus / Trash2` 共 10+ 装饰性图标 | 每个 setting 都拖一个图标，视觉噪声严重，污染信息层级 |
| 信息组织 | "Current agent" 卡同时塞了 role 文件 / 模型卡 / 4 个 toggle / 3 个 dropdown / temperature slider，全部混在同一可折叠区里 | 缺少"按职责分组"，扫读困难 |
| 文件查看 | SOUL / IDENTITY / USER / MEMORY / AGENTS / TOOLS 在二级面板里通过 tab 切换 | 切换文件时丢失上下文，没有共览性 |

## 2. 设计目标（对应 kira 提出的三条标准）

| kira 标准 | 本规范要求 |
|-----------|------------|
| (1) 没有二级页面 | 删除 `isContextPanelOpen` / `isModelPanelOpen` 两套覆盖层。Agent profile 与 kernel 文件编辑全部转为**就地行内展开**（accordion）。整个抽屉只有一个纵向滚动视图。 |
| (2) 没有气泡、不被图标污染、追求极简工程风 | 取消所有"卡片容器"。分区之间只用 1px 横线 + 小号 uppercase 章节名分隔。彻底删除装饰性图标，只保留**结构性图标**：`ChevronRight/Down`（展开标识）与 `X`（关闭抽屉）。 |
| (3) 合理规划可配置项位置 | 按 **agent 身份 → 采样 → 工具 → 推理 → 上下文 → 内核 → 记忆运行时** 七个分区线性排列，每个分区只承担一类职责（与 20260512-2 的"职责硬分隔"原则同源）。 |

## 3. 视觉语言

### 3.1 容器与分隔

- **不再使用卡片**。抽屉内部是单一纵向 list，宽度沿用 `22.5rem`。
- **分区之间**用 `border-t border-border pt-4 mt-6` 实现的细横线分隔；没有 border-radius，没有 box-shadow，没有 surface-alt 填充。
- **抽屉自身**只保留左侧细分隔线 `border-l border-border`，背景与主对话区一致（`bg-surface-alt` 维持，但去掉所有内部 surface-alt 叠加）。

### 3.2 章节标题

```
SAMPLING          ← 12px、tracking-[0.12em]、uppercase、text-text-muted、font-medium
─────────────────  ← border-t border-border，section 上方
```

不使用 emoji，不使用装饰图标，不使用 `h3 text-[18px] font-semibold`（当前抽屉里 "Current agent" / "Lecquy kernel" 的样式）。

### 3.3 设置行

每行只有两种形态：

1. **label + 控件**（一行）：左侧标签 `text-sm text-text-primary`，右侧控件右对齐。
2. **label + 副文 + 控件**（两行内）：第一行 label，第二行 `text-xs text-text-muted` 副文，控件在第一行右侧。

行间距统一 `py-2.5`；行之间不加底色，不加 border，不加 hover 高亮（除非可点击展开）。

### 3.4 控件样式

| 控件 | 样式 |
|------|------|
| 文本输入 | 单行 `border-b border-border bg-transparent py-1.5 text-sm` —— **下划线式**，不再是 `rounded-xl border + bg-surface-alt + shadow` 的胶囊框 |
| 数字输入 / slider | slider 用细单线轨道，accent 用 `text-text-primary`；右侧显示数值（不再用外框数字输入框） |
| 下拉 | inline `button` 文本 `text-sm` + `ChevronDown size-3.5`，点击在下方就地展开 menu（无 backdrop、无圆角阴影，只有 1px 边） |
| 切换开关 | 保留当前 toggle pill（极小幅度调整：去掉 border，只留 `bg-toggle-on/off`），高度 `h-5` 而非 `h-6` |
| 展开/折叠 | 标题行左侧 `ChevronRight`，展开后变 `ChevronDown`，内容**直接缩进 12px** 在下方展开，不另设容器边框 |

### 3.5 图标策略

- **允许**：`ChevronRight`、`ChevronDown`、`X`、`Plus`（新增 agent，只在一个位置出现）、`Trash2`（删除 agent，只在一个位置出现）。
- **删除**：`Bot`、`Brain`、`Cpu`、`Database`、`FileText`、`SlidersHorizontal`、`Wrench`、`Pencil`。

> 工程风的核心是"文字承担信息"，图标只用于动作语义不可省略的位置。

### 3.6 颜色

只用三档文字颜色：

- `text-text-primary` —— label、当前值、激活态
- `text-text-secondary` —— 默认 setting 副文
- `text-text-muted` —— 章节名、提示性数值（如 "8k"）、文件路径

警告条仅保留一处（`memory.db` 衰减提示），改为单行 `text-xs text-amber-600` + 左侧细竖条，去掉黄色填充背景。

### 4.1 AGENT 分区

- 替代当前的 "Current agent" 折叠卡 + Agent profile 二级面板。
- 5 行直接 inline：
  1. **Preset**：下拉切换已有 agent；右侧紧跟 `+`（新建）与 `⌫`（删除当前）。`+` 触发**就地切换到 "untitled" 草稿**，不弹二级面板。
  2. **Name** / **Model** / **Base URL** / **API Key**：单行下划线输入框；250ms 防抖自动落盘（沿用现有 `modelSaveStatus === 'Editing'` 自动保存机制）。
- **不再有** "连接" 按钮——`Base URL` 失焦时自动调用 `fetchModelName`（如果 `Model` 为空），失败时在分区底部显示一行 `text-xs text-red-600` 错误。
- **删除** "Agent profiles are saved in local storage" 这种 footnote。

### 4.2 SAMPLING 分区

- 只保留 `temperature` slider（无右侧数字输入框，直接在 label 行右端显示当前值）。
- `max tokens` 改为 inline 下拉，右侧附带 hint（`8k` / `16k` / `32k`），不再独立成行。

### 4.3 TOOLS 分区

- 只放 `Function calling` 一项。理由：tools 的纪律已经收敛到 `.lecquy/TOOLS.md`（20260512-2 §3），UI 不需要重复展示工具 inventory。

### 4.4 REASONING 分区

- 把"Thinking enabled / Thinking protocol / Thinking level"三项合并到一个分区，与 SAMPLING 平级。
- 当 `protocol === 'off'` 时，`Enabled` 与 `Level` 行整体降低透明度（`opacity-50` + `pointer-events-none`），不要禁用单个控件后让 layout 漂浮。

### 4.5 ROLE CONTEXT 分区

- 来自当前 `activeAgentRoleFiles`（默认 `Role.md` / `Tools.md`）。
- 每行 `▸ Role.md`，点击就地展开 inline `<pre>` 显示内容（只读）；再次点击折叠。
- **不再** 用 2 列 grid + chip 样式。

### 4.6 KERNEL 分区

- 取代当前的 "Lecquy kernel" 折叠卡 + Context 二级面板。
- 6 行可展开列表（顺序与 20260512-2 §3 文件表一致）：
  - 前 4 个（`SOUL.md` / `IDENTITY.md` / `USER.md` / `MEMORY.md`）展开后是 inline `<textarea>`，250ms 防抖自动落盘。
  - 后 2 个（`AGENTS.md` / `TOOLS.md`）展开后是只读 `<pre>`。
- label 右侧用 `text-xs text-text-muted` 显示职责描述（一句话）；不再用 chip 显示路径——路径只在展开后的 footer 用 `<code class="text-[11px] text-text-muted">.lecquy/SOUL.md</code>` 显示。
- 一次只允许一个文件展开（单选 accordion）；展开新文件时先保存旧文件草稿（沿用现有 `persistContextFile`）。

### 4.7 MEMORY RUNTIME 分区

- 来自当前 MEMORY.md 二级面板里夹带的 "Memory runtime" 与 logs 列表，单独提到顶层分区。
- 4 项：
  1. **Embedding URL** 输入框
  2. **Flush turns** 数字输入
  3. **memory.db** 只显示一行 `SQLite · {size}`，不再有"查看/编辑"按钮（编辑入口是上方 `MEMORY.md`）
  4. **Logs** 行可展开，展开后是只读文件列表（沿用现有 `memoryFiles`）
- 分区末尾显示一行衰减警告（去掉黄色填充，改用左侧 1px 竖线 + amber 文字）。

## 5. 字段 → pi-agent-core 参数映射

确认所有暴露给用户的可配置项都对应 pi-agent-core 标准支持的参数：

| UI 字段 | pi-agent-core 字段 | 备注 |
|---------|-------------------|------|
| Agent · Model | `config.model` | 必填 |
| Agent · Base URL | （Lecquy 自带，不在 agentLoop 直接参数列表） | 通过 `config.transport` / `config.headers` 间接生效；当前由 ModelConfig 透传 |
| Agent · API Key | `config.apiKey` / `config.getApiKey` | 二选一 |
| Sampling · Temperature | `config.temperature` | 0–2 |
| Sampling · Max tokens | `config.maxTokens` | 三档预设映射 |
| Tools · Function calling | `context.tools` 是否注入 | 关闭时 builder 不写工具 schema |
| Reasoning · Enabled | `config.reasoning.enabled` | 通过 `config.reasoning` |
| Reasoning · Protocol | `config.reasoning.protocol` | qwen / zai / openai\_reasoning / off |
| Reasoning · Level | `config.reasoning.level` 或 `config.thinkingBudgets` | level 与 budget 通过 protocol adapter 转换 |

未暴露在设置栏的（属于内部运行时，不应该让用户配置）：`onPayload` / `headers` / `cacheRetention` / `sessionId` / `transport` / `maxRetryDelayMs` / `metadata` / `convertToLlm` / `transformContext` / `getSteeringMessages` / `getFollowUpMessages` / `signal` / `streamFn`。

## 6. 交互细节

### 6.1 展开/折叠

- 所有 accordion 项的点击命中区是**整行**（不限于 chevron），通过 `aria-expanded` 标识状态。
- 展开动画：仅 `transition-[grid-template-rows]` 0.2s ease，不要弹性曲线。
- 同一分区内允许多个 accordion 同时展开（如 Role context 的 Role.md + Tools.md 可同开）；**KERNEL 分区是例外，单选**——因为同时编辑两个文件会让自动保存语义混乱。

### 6.2 保存状态

- 不再在面板底部显示 "Saved / Editing" 文案。
- 输入框失焦或 250ms 防抖触发保存；保存失败时在该字段下方显示一行 `text-xs text-red-600` 错误，保存成功**不显示** "已保存"——静默成功是工程风的本色。

### 6.3 抽屉宽度

- 沿用 `22.5rem`。
- 抽屉打开/关闭动画沿用现有 `width 0 ↔ 22.5rem` 方案，不改动主对话区布局。

### 6.4 滚动条

- 沿用现有 `settings-scrollbar-hidden` 类。
- 顶部 mask 渐变沿用，避免内容直接撞顶。

## 7. 不做什么

- 不引入 dark/light 主题切换按钮（已在抽屉外顶栏存在）。
- 不引入"高级/简单"模式切换。
- 不引入 plan / manager / worker mode 切换按钮（路由在主对话区上方，不在设置栏）。
- 不引入 plugins / MCP servers / skills 列表（按 CLAUDE.md §2.2 延后清单）。
- 不引入 `convertToLlm` / `transformContext` 等运行时钩子的 UI 配置入口（这些是代码层接口，不是用户配置项）。

| 检查项 | 通过标准 |

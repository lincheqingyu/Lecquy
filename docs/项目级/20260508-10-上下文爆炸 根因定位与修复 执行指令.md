## 一、根因（已锁定，不需要重新排查）

`backend/src/agent/tools/session-tools/sessions-history.ts` 的 `sessions_history` 工具有三个叠加缺陷：

1. **`limit` 是可选的**：agent 不传则全量返回所有 entries
2. **JSON 输出无截断**：直接 `JSON.stringify(rows, null, 2)`，不走 `TOOL_OUTPUT_LIMIT` 50K 字符上限
3. **不识别 `current` 关键字**：agent 传 `current` 时 `service.history('current')` 报"会话不存在"

历史会话文件实测大小：

```
.lecquy/sessions/v3/sessions/sess_1776231258671_*.jsonl    5.2 MB
.lecquy/sessions/v3/sessions/sess_1778293348891_*.jsonl    3.4 MB    ← 单这一个就够把 1M 上下文炸 3 倍
.lecquy/sessions/v3/sessions/sess_1777271961091_*.jsonl    1.0 MB
```

任何一次 agent 调 `sessions_history(<sessionKey>)` 不带 limit，都可能触发 1M+ token 上下文爆炸。

## 五、动手清单

### 5.1 修 `sessions-history.ts`

**改造目标**：

| 项 | 改造前 | 改造后 |
|---|---|---|
| `limit` | optional, max=500 | **default=20**, max=200（向下收紧） |
| 输出截断 | 无 | 走 `TOOL_OUTPUT_LIMIT` 50K 字符截断 |
| `current` 关键字 | 报错 | 解析为当前 runtime session |
| 工具描述 | "读取指定会话的历史消息。" | 加上"limit 默认 20 条；调用 sessions_list 先看可用会话；不要传 'current'" |

**实现要点**：

```ts
import { Type } from '@sinclair/typebox'
import type { AgentTool, AgentToolResult } from '@mariozechner/pi-agent-core'
import { TOOL_OUTPUT_LIMIT } from '../../types.js'
import { getBoundSessionService, getCurrentSessionKey } from './runtime.js'

const DEFAULT_HISTORY_LIMIT = 20
const MAX_HISTORY_LIMIT = 200   // 上限收紧（原来 500）

const parameters = Type.Object({
  sessionKey: Type.String({
    minLength: 1,
    description: '目标会话的 session key。如需当前会话，使用此次调用所在的 sessionKey 而不是 "current"。',
  }),
  limit: Type.Optional(Type.Number({
    minimum: 1,
    maximum: MAX_HISTORY_LIMIT,
    default: DEFAULT_HISTORY_LIMIT,
    description: `返回最近 N 条 entries。不传则取 ${DEFAULT_HISTORY_LIMIT}。`,
  })),
})

export function createSessionsHistoryTool(): AgentTool<typeof parameters> {
  return {
    name: 'sessions_history',
    label: '读取会话历史',
    description:
      '读取指定会话最近 N 条历史 entries。' +
      '\n用法：' +
      `\n- limit 默认 ${DEFAULT_HISTORY_LIMIT} 条，最多 ${MAX_HISTORY_LIMIT} 条；` +
      '\n- 调 sessions_list 先看有哪些可用会话；' +
      '\n- 不要传 sessionKey="current"——必须传具体的 session key。' +
      `\n- 单次返回输出会被截断到 ${TOOL_OUTPUT_LIMIT} 字符以避免上下文爆炸。`,
    parameters,
    execute: async (_id, params): Promise<AgentToolResult<Record<string, never>>> => {
      // 处理 'current' 关键字：解析为当前会话
      let sessionKey = params.sessionKey
      if (sessionKey === 'current' || sessionKey === '__current__') {
        const current = getCurrentSessionKey()
        if (!current) {
          return {
            content: [{ type: 'text', text: JSON.stringify({
              error: 'sessionKey="current" 不可用：当前没有绑定的运行时会话。请改传具体 session key。',
            }) }],
            details: {},
          }
        }
        sessionKey = current
      }

      const limit = params.limit ?? DEFAULT_HISTORY_LIMIT
      const rows = await getBoundSessionService().history(sessionKey, limit)

      // 强制截断到 TOOL_OUTPUT_LIMIT 字符
      const fullJson = JSON.stringify(rows, null, 2)
      const truncated = fullJson.length > TOOL_OUTPUT_LIMIT
      const text = truncated
        ? fullJson.slice(0, TOOL_OUTPUT_LIMIT) + `\n\n... [输出被截断，原始 ${fullJson.length} 字符 > 上限 ${TOOL_OUTPUT_LIMIT}。请用更小的 limit。]`
        : fullJson

      return {
        content: [{ type: 'text', text }],
        details: {},
      }
    },
  }
}
```

### 5.2 在 `runtime.ts` 暴露 `getCurrentSessionKey`

如果 `runtime.ts` 已经维护了"当前 runtime session"上下文，加一个 export：

```ts
// runtime.ts
let currentSessionKey: string | null = null

export function setCurrentSessionKey(key: string): void { currentSessionKey = key }
export function clearCurrentSessionKey(): void { currentSessionKey = null }
export function getCurrentSessionKey(): string | null { return currentSessionKey }
```

并确保在 `simple-handler` / `plan-handler` 启动会话时调 `setCurrentSessionKey`。如果实现复杂，本次先**返回 null**（也就是 `current` 关键字暂时无解析），让错误信息提示 agent 用真实 key——这比无脑 fallback 安全。

## 一、背景与立场（这次审查的"判断标尺"）

项目作者 kira 正在和另一个 AI 助手 (Claude) 重新定位 Lecquy，并已得出明确结论：

**新定位**：
> 一个**只服务作者一人**、**深度懂他**的编程协作者，且**能随他演化**。

定位的反义词是 Codex / Claude Code（"陌生人都能用的通用工具"）。
定位的近义词是 JARVIS（独占性 + 长期记忆）+ hermes-agent（自我进化）。
作者最高频场景是**编程**，但他要的不是"另一个编程助手"，而是"一个跟他共生了一年的搭档"。

**产品命脉优先级**（重要性递减）：

1. **记忆** —— 决定"懂我"
2. **自我进化** —— 决定"会变"
3. **上下文工程 + 循环** —— 上面两者的载体
4. **工程债**（沙箱、数据库、压缩、WS 重构）—— 不出事就行
5. **暂时推迟**：RAG / websearch / 语音 / 任务模式 / 插件系统 / 鉴权 / 限流 / 多用户 Gateway

**你（DeepSeek）这次审查的任务，就是按这把尺子，给每块现有代码做事实定级**：
是真的在跑、还是只是占位？
属于"命脉"路线、还是"通用 Agent 框架"路线？

## 三、必须逐条回答的 7 个核心问题

### 问题 1：记忆系统是不是真的？

回答以下子问题：

- `memory/` 模块**实际写入磁盘**的是什么内容？用了哪个目录、哪种格式（JSON / Markdown / SQLite / 纯文本）？
- "flush" 的具体动作是：直接 dump 全部对话、还是做了抽取/压缩/分类/打标签？给出 flush 触发条件（每 N 轮？某事件？）。
- **新对话开始时**，记忆是怎么被注入到上下文的？是否注入？注入到 system prompt、第一条 user message、还是其他位置？
- **判断题**：当前的记忆能否支持"我两周前在 X 项目用过 Y 解法"这类长期检索？给出代码层面的证据（有索引吗？有时间戳吗？有项目维度吗？）。

### 问题 2：一次请求的上下文是怎么拼出来的？

完整跟踪一次 chat 请求，写出**精确的拼装顺序**：

```
[用户输入] → [系统提示组装] → [历史对话注入] → [工具描述] → [记忆注入?] → [项目上下文注入?] → [调用 LLM]
```

每一步要回答：

- 内容**从哪个文件 / 哪个变量来**
- 是否有截断逻辑、截断阈值、截断策略（保头？保尾？摘要替换？）
- 是否注入了"项目级上下文"（如 `AGENTS.md`、`CLAUDE.md`、当前 cwd 标识）
- **关键判断**：当前有没有"个人化注入点"（user identity / 偏好 / 历史决策摘要）？如果没有，最自然的**新增注入点**是哪个文件的哪几行？

### 问题 3：主循环和子 Agent 现在到底在跑什么？

- `runMainAgent` 单次调用最多跑多少 turn？终止条件是什么（无 tool_call？hit 上限？显式 done？）
- 子 Agent / worker 的**触发条件**、**上下文是否继承自主 Agent**、**结果如何回写主上下文**
- `.claude/backend/architecture_and_dev_plan.md` 描述的"Plan 模式 / Manager-Worker"是已落地、半落地、还是仅文档？请明确——只看代码，不看文档承诺。

### 问题 4：压缩做到哪儿了？

- 全仓搜索 `compress` / `summarize` / `compact` / `truncate` 关键字，列出所有相关代码位置
- 区分：真在运行的 / 已写但未挂接的 / 占位空函数 / 注释掉的
- 当前唯一在生效的"上下文瘦身"机制是什么？阈值多少？

### 问题 5：自我进化的"接口"现在能不能开？

新定位要求 agent 能自己改自己的 prompt / skill / 记忆结构。请判断：

- **写文件工具**（`write_file`、`edit_file`、`bash`）当前是否能写入这些"自身配置"路径：`.claude/`、`.lecquy/`、`backend/src/core/prompts/`、`memory/`、`skills/`？是有显式拦截、还是默认放行？
- Skill 系统是**热加载**（运行时改文件下次对话立刻生效）还是**冷加载**（改完必须重启）？给出 `SkillLoader` 加载时机的代码证据。
- 系统提示能否在运行时被替换 / 拼接外部文件？还是固化在编译产物（如打包后的 `dist/`）里？
- 是否存在任何**"agent 自我反思"**的痕迹（让 agent 看自己上一轮表现、写回某文件）？哪怕是半成品也要列出。

### 问题 6：沙箱实际能不能用？

- `bash` 工具当前隔离方式：直接 `execSync` / `spawn`？有 cwd 锁定吗？有路径白名单/黑名单吗？环境变量是否清理？
- 文档（`architecture-evolution.md`）说"沙箱做了一半，无法测试"——具体卡在哪一步？是某个抽象没写完？测试环境缺失？还是设计本身没合上？

### 问题 7：哪些代码明显是"通用 Agent 框架"路线的产物？

按"独占性 vs 通用化"的视角清查：

- 列出所有跟**多用户 / 鉴权 / 限流 / Gateway / 多 session 隔离 / CORS 收紧 / token 校验**相关的文件
- 估算这部分代码占 `backend/src/` 总行数的百分比（粗估即可）
- **判断**：如果今天就承认"只给一个人用"，这些代码哪些可以**直接删除**（不影响命脉功能）？哪些是**已经写但还没启用**（决定权还在配置）？哪些是**已嵌入主路径必须改**？

## 二、目标（一句话）

把 Lecquy 后端"通用 Agent 框架方向"产出的废弃/未启用代码物理删除，让代码库瘦身约 5~10%，让接下来动手的人视野不被无效代码污染。

---

## 三、不要做什么

- **不要**重构任何还在跑的代码——本次只删，不改
- **不要**修改任何配置文件（`.env` / `package.json` / `tsconfig.json`）
- **不要**触碰 `memory/`、`runtime/permissions/sandbox-adapter.ts`、`agent/agent-runner.ts`、`ws/chat-ws.ts` 这几个核心文件
- **不要**删除 `db/` 目录（PG 留待后续单独决策，本周不动）
- **不要**删除 `runtime/permissions/` 整个目录（沙箱有用，本周第 2 件要接它）
- 删除任何文件之前**必须先 grep 它的引用**，发现引用就**停下来在指令底部 "执行报告" 段记录，不要自作主张修改引用方**

## 四、删除清单（按顺序执行）

### 4.1 删除 HTTP 旧路由

**文件**：`backend/src/controllers/chat.ts`（16 行，仅返 410 Gone）

**删除前检查**：
```bash
grep -rn "controllers/chat\|chatRouter" backend/src --include="*.ts"
```

**预期引用方**：`backend/src/app.ts`（注册路由的地方）。

**动作**：
1. 删除文件 `backend/src/controllers/chat.ts`
2. 在 `backend/src/app.ts` 中删除 `import { chatRouter }` 一行 和 `app.use(...chatRouter...)` 一行
3. 如 `controllers/` 目录删完为空则把目录也删掉

**验收**：`pnpm -F @lecquy/backend build` 能通过。

---

### 4.2 删除旧 Session 模块（已被 session-v2 替代）

**文件**：
- `backend/src/session/session-registry.ts`（约 5.6 KB）
- `backend/src/session/session-state.ts`（约 3.8 KB）
- `backend/src/session/index.ts`（284 字节，barrel 导出）

**删除前检查**：
```bash
grep -rn "from.*['\"].*session/session-registry\|from.*['\"].*session/session-state\|from.*['\"].*\\./session['\"]" backend/src --include="*.ts"
```

**预期**：被 session-v2 替代，主路径不应再引用。如发现 `ws/chat-ws.ts` 等关键文件仍在 import，**立刻停下报告**，不要继续。

**动作**：
1. 删除三个文件
2. 删除空目录 `backend/src/session/`

**验收**：`pnpm -F @lecquy/backend build` 能通过。

---

### 4.3 砍掉多 Session 协作工具（单人场景无用武之地）

**文件**：
- `backend/src/agent/tools/session-tools/sessions-spawn.ts`
- `backend/src/agent/tools/session-tools/sessions-send.ts`
- `backend/src/agent/tools/session-tools/sessions-list.ts`
- `backend/src/agent/tools/session-tools/sessions-history.ts`
- `backend/src/agent/tools/session-tools/runtime.ts`
- `backend/src/agent/tools/session-tools/index.ts`

**删除前检查**：
```bash
grep -rn "createSessionsSpawn\|createSessionsSend\|createSessionsList\|createSessionsHistory\|session-tools" backend/src --include="*.ts"
```

**预期引用方**：`backend/src/agent/tools/index.ts`（在 `createSimpleTools` / `createManagerTools` / `createWorkerTools` 中注册了这些工具）。

**动作**：
1. 修改 `backend/src/agent/tools/index.ts`：删除四个 `import { createSessionsXxxTool }` 行，删除 `createSimpleTools` / `createManagerTools` 数组中对应的 `createSessionsXxxTool()` 调用
2. 删除整个 `backend/src/agent/tools/session-tools/` 目录

**验收**：
- `pnpm -F @lecquy/backend build` 能通过
- 启动后端、跑一次 simple 模式对话，不报"工具不存在"错

---

### 4.4 清理 LEGACY .ZxhClaw 引用（仅源码层面，不动 release / worktrees）

**仅处理**：`backend/src/core/runtime-paths.ts` 中将 `.ZxhClaw` 列入 `LEGACY_ROOT_RUNTIME_DIR_NAMES` 的代码（如还存在）。

**不要动**：
- `.claude/worktrees/*/` 下的 `.ZxhClaw/`（git worktree 目录，与主分支无关）
- `release/portable/.ZxhClaw/`、`release/sea/*/.ZxhClaw/`（打包产物，build 时会重生成）

**动作**：在 `runtime-paths.ts` 里搜索 `ZxhClaw`，如果只是兼容旧目录的 fallback 逻辑且没有真实数据依赖，可以删除该 fallback。如果发现**当前还有真实依赖**则停下报告。

---

### 4.5 删除 Plan 模式幽灵代码（可选，把握不准则跳过）

**背景**：DeepSeek 审查报告指出 `WorkerReceipt` 类型、Manager/Worker 工具白名单、`prompt-layer-types.ts` 中相关定义是孤儿代码——`sub-agent-runner.ts` 不存在、没有执行器把它们串起来。

**条件触发**：仅当你能在 5 分钟内确认"这些类型/常量在生产路径无任何引用"时才删；否则跳过留给下次。

**候选清理点**：
- `backend/src/agent/types.ts` 中 `MAX_ITERATIONS`、`MAX_TOOL_FAILURES` 之外的 Manager/Worker 常量（如有）
- `backend/src/core/prompts/prompt-layer-types.ts` 中 `WorkerReceipt` 类型
- `backend/src/agent/tool-permission.ts` 中 Manager/Worker 白名单（如代码中实际从未引用 `role: 'manager'` 或 `role: 'worker'`）

**安全原则**：宁愿跳过，也不要漏删导致 TS 编译断裂。

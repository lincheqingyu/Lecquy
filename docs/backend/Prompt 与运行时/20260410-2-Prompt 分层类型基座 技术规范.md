# Prompt 分层类型基座 技术规范

更新日期：2026-04-10

所属开发包：包 0（类型基座）
前置依赖：无
预估工期：0.5 天

## 1. 目标

把 v2 文档的 7 层分层模型、capability block、USER.md schema、worker 回执、权限三档变成 TypeScript 类型定义，供后续所有开发包引用。

本包是纯类型变更，不改变任何运行时行为，零风险合入。

## 2. 新建文件

路径：`backend/src/core/prompts/prompt-layer-types.ts`

## 3. 类型定义

### 3.1 分层枚举

```typescript
/**
 * Prompt 7 层优先级枚举
 * 数值 = 优先级顺序 = 注入顺序
 * 1-5 为 prefix cache 段，6-7 为动态段
 */
export enum PromptLayer {
  System = 1,
  Mode = 2,
  StartupContext = 3,
  SkillRuntime = 4,
  UserPreference = 5,
  MemoryRecall = 6,
  LiveTurn = 7,
}
```

### 3.2 LAYER 标签常量

```typescript
/**
 * 与 PromptLayer 一一对应的 LAYER 标签名
 * 标签名是字节级契约，禁止修改
 */
export const LAYER_TAGS = {
  [PromptLayer.System]: 'system',
  [PromptLayer.Mode]: 'mode',
  [PromptLayer.StartupContext]: 'startup',
  [PromptLayer.SkillRuntime]: 'skill',
  [PromptLayer.UserPreference]: 'user_preference',
  [PromptLayer.MemoryRecall]: 'memory_recall',
  [PromptLayer.LiveTurn]: 'live_turn',
} as const satisfies Record<PromptLayer, string>

/**
 * prefix cache 段的层级范围
 */
export const PREFIX_CACHE_LAYERS = [
  PromptLayer.System,
  PromptLayer.Mode,
  PromptLayer.StartupContext,
  PromptLayer.SkillRuntime,
  PromptLayer.UserPreference,
] as const

/**
 * 动态段的层级范围
 */
export const DYNAMIC_LAYERS = [
  PromptLayer.MemoryRecall,

```

### 3.3 单层切片

```typescript
/**
 * 序列化器的输入单元
 */
export interface LayerSlice {
  /** 所属层级 */
  layer: PromptLayer
  /** LAYER 标签名 */
  tag: string
  /** 该层的完整文本内容 */
  content: string
  /** 内容的 SHA256 哈希，用于 cache 命中判断 */
  contentHash: string
  /** 粗估 token 数（按 1 token ≈ 3.5 字符估算） */
  tokenEstimate: number
}
```

### 3.4 Capability Block

```typescript
/**
 * 运行时能力声明
 * 逻辑属于 system 层，物理注入在 startup 段首部
 */
export interface CapabilityBlock {
  /** 当前 OS 执行器 */
  executor: 'powershell' | 'shell' | 'none'
  /** 可用能力列表 */
  available: string[]
  /** 不可用能力列表 */
  unavailable: string[]
}
```

### 3.5 USER.md 切片

```typescript
/**
 * USER.md 解析产物
 */
export interface UserMdSlices {
  /** profile 段内容 → 注入层 3 startup context */
  profileSlice: string
  /** preference 段内容 → 注入层 5 user preference */
  preferenceSlice: string
  /** 是否被拒绝（第三个 H2、schema 不匹配等） */
  rejected: boolean
  /** 拒绝原因 */
  rejectReason?: string
}

/**
 * USER.md 的 frontmatter schema 版本
 */
export const USER_MD_SCHEMA = 'lecquy.user/v1' as const
```

### 3.6 Worker 回执

```typescript
/**
 * worker 完成单个 todo 后必须返回的结构化回执
 */
export interface WorkerReceipt {
  /** 执行结果 */
  result: string
  /** 验证说明 */
  validation: string
  /** 对下一步的建议（可选） */
  nextHint?: string
}
```

### 3.7 权限三档

```typescript
/**
 * 工具调用权限等级
 * auto: 直接执行
 * preamble: 执行前发 ≤1 句说明，不等待确认
 * confirm: 必须显式确认
 */
export enum PermissionTier {
  Auto = 'auto',
  Preamble = 'preamble',
  Confirm = 'confirm',
}
```

### 3.8 Agent 角色

```typescript
/**
 * Agent 角色枚举（与当前 system-prompts.ts 中的 role 字段对齐）

export type AgentRole = 'simple' | 'manager' | 'worker'
```

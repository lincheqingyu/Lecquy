## 3. 新建 `agent/tool-permission.ts`

### 3.1 权限三档定义

| 档位 | 范围 | 默认策略 |
|---|---|---|
| `auto` | 只读命令、workspace 内新建与追加、skill、session 查询 | 直接执行 |
| `preamble` | 跨多文件编辑、长耗时命令、workspace 内覆盖写 | 执行前发 ≤1 句中文说明，不等待确认 |
| `confirm` | workspace 外写、带副作用系统命令、安装依赖、部署、外部状态修改 | 必须显式确认 |

### 3.2 核心函数

```typescript
import { PermissionTier, AgentRole } from '../core/prompts/prompt-layer-types'

/**
 * 判定单次工具调用的权限等级
 */
export function classifyToolPermission(
  toolName: string,
  args: Record<string, unknown>,
  role: AgentRole,
  workspaceDir: string
): PermissionTier
```

### 3.3 分类规则

```typescript
function classifyToolPermission(toolName, args, role, workspaceDir): PermissionTier {
  // ---- auto 档 ----
  // read_file: 始终 auto
  if (toolName === 'read_file') return Auto

  // skill: 始终 auto
  if (toolName === 'skill') return Auto

  // sessions_list, sessions_history: 始终 auto
  if (['sessions_list', 'sessions_history'].includes(toolName)) return Auto

  // ---- write_file / edit_file ----
  if (['write_file', 'edit_file'].includes(toolName)) {
    const filePath = args.path as string
    const isInWorkspace = filePath.startsWith(workspaceDir)

    if (!isInWorkspace) return Confirm  // workspace 外 → confirm

    // workspace 内：检查文件是否已存在
    // 新建/追加 → auto
    // 覆盖已有文件 → preamble
    if (fileExists(filePath)) return Preamble
    return Auto
  }

  // ---- bash ----
  if (toolName === 'bash') {
    const command = args.command as string

    // 副作用关键词检测
    const CONFIRM_PATTERNS = [
      /\brm\s/, /\brmdir\b/, /\bdel\b/, /\bRemove-Item\b/,
      /\binstall\b/, /\buninstall\b/,
      /\bdeploy\b/, /\bpush\b/,
      /\bchmod\b/, /\bchown\b/,
      /\bsystemctl\b/, /\bservice\b/,
      /\bkill\b/, /\bpkill\b/,
      /\bdrop\s/, /\btruncate\s/, /\bdelete\s+from\b/i,
    ]

    if (CONFIRM_PATTERNS.some(p => p.test(command))) return Confirm

    // 长耗时或多文件操作 → preamble
    const PREAMBLE_PATTERNS = [
      /\bfind\b.*-exec/, /\bxargs\b/,
      /\bgrep\b.*-r/, /\bsed\b.*-i/,
      /\bwget\b/, /\bcurl\b.*-o/,
    ]

    if (PREAMBLE_PATTERNS.some(p => p.test(command))) return Preamble

    // 其他 bash 命令（ls, cat, echo, pwd 等）→ auto
    return Auto
  }

  // ---- todo_write ----
  if (toolName === 'todo_write') return Auto

  // ---- request_user_input ----
  if (toolName === 'request_user_input') return Auto

  // ---- sessions_send, sessions_spawn ----
  if (toolName === 'sessions_send') return Preamble
  if (toolName === 'sessions_spawn') return Confirm

  // ---- 未知工具 → confirm（安全兜底）----
  return Confirm
}
```

### 3.4 角色白名单函数

```typescript
/**
 * manager 允许使用的工具白名单
 * manager 默认只有 auto 档，不持有 preamble / confirm 档能力
 */
export function isManagerAllowed(toolName: string): boolean {
  const MANAGER_WHITELIST = [
    'read_file',
    'skill',
    'todo_write',
    'request_user_input',
    'sessions_list',
    'sessions_history',
    'sessions_send',
  ]
  return MANAGER_WHITELIST.includes(toolName)
}

/**
 * worker 允许使用的工具
 * 与 simple 一致，但移除 todo_write 和 sessions_spawn
 */
export function isWorkerAllowed(toolName: string): boolean {
  const WORKER_BLACKLIST = ['todo_write', 'sessions_spawn']
  return !WORKER_BLACKLIST.includes(toolName)
}
```

```typescript
export function createManagerTools(todoManager: TodoManager): AgentTool[] {
  const allTools = [
    readFileTool,
    skillTool,
    todoWriteTool(todoManager),
    requestUserInputTool,
    ...sessionTools,
  ]
  // 强制过滤：只保留白名单内的工具
  return allTools.filter(t => isManagerAllowed(t.name))
}

export function createWorkerTools(): AgentTool[] {
  const allTools = createSimpleTools()
  // 移除 todo_write 和 sessions_spawn
  return allTools.filter(t => isWorkerAllowed(t.name))
}
```

## 5. 改造 `agent/agent-runner.ts`

### 5.1 权限三档接入

在 `agentLoop` 的工具调用执行前插入权限判定：

```typescript
// agentLoop 中，每次 LLM 返回工具调用时
for (const toolCall of toolCalls) {
  const tier = classifyToolPermission(
    toolCall.name,
    toolCall.arguments,
    options.role ?? 'simple',
    options.workspaceDir
  )

  switch (tier) {
    case PermissionTier.Auto:
      // 直接执行
      await executeTool(toolCall)
      break

    case PermissionTier.Preamble:
      // 发 preamble 事件（≤1 句说明）后执行
      options.onEvent?.({
        type: 'preamble',
        toolName: toolCall.name,
        // 由 LLM 在调用前已产生的说明文本
      })
      await executeTool(toolCall)
      break

    case PermissionTier.Confirm:
      // 发 confirm 事件，暂停等待用户确认
      options.onEvent?.({
        type: 'confirm_required',
        toolName: toolCall.name,
        args: toolCall.arguments,
        description: generateConfirmDescription(toolCall),
      })
      // 等待用户通过 WS 确认或拒绝
      const approved = await waitForUserConfirmation(toolCall.id)
      if (approved) {
        await executeTool(toolCall)
      } else {
        // 返回工具执行失败，说明用户拒绝
        await returnToolError(toolCall.id, '用户拒绝了此操作')
      }
      break
  }
}
```

### 5.2 preamble 生成

preamble 不是由 agent-runner 自行生成文案的，而是利用 LLM 在工具调用前自然产生的说明文本。agent-runner 只负责：

1. 检测到 preamble 档的工具调用
2. 将 LLM 已输出的文本片段作为 preamble 发送给前端
3. 然后执行工具

如果 LLM 没有在调用前产生说明，agent-runner 生成一条默认说明：

```typescript
function generateDefaultPreamble(toolCall: ToolCall): string {
  return `正在执行 ${toolCall.name}...`
}
```

## 6. 改造 `agent/manager-runner.ts`

### 6.1 当前状态

- 工具集：read_file + skill + todo_write + request_user_input + session tools
- 最大迭代：MAX_ITERATIONS = 10

### 6.2 改造要点

1. **工具集不变**（已经是只读 + todo_write），但在文档注释中明确 manager 的权限等级：

```typescript
/**
 * Manager Agent
 *
 * 权限等级：仅 auto 档
 * 工具集：read_file, skill, todo_write, request_user_input, session 查询工具
 * 禁止：bash, write_file, edit_file, sessions_spawn
 * 不直接执行带副作用操作，所有副作用通过授权 worker 执行
 */
```

2. **处理 worker 回执**：

```typescript
// manager 收到 worker 回执后的决策逻辑
function handleWorkerReceipt(receipt: WorkerReceipt, todoId: string): ManagerDecision {
  if (receipt.result === 'blocked') {
    // worker 连续失败，manager 决策：拆分或标记 blocked
    return { action: 'split' | 'block', todoId }
  }
  // 正常完成
  return { action: 'complete', todoId, nextHint: receipt.nextHint }
}

type ManagerDecision =
  | { action: 'complete'; todoId: string; nextHint?: string }
  | { action: 'retry_with_change'; todoId: string; newPrompt: string }
  | { action: 'split'; todoId: string; subTodos: string[] }
  | { action: 'block'; todoId: string; reason: string }
```

3. **worker confirm 回交**：当 worker 遇到 confirm 档操作时，不直接向用户询问，而是返回 blocked receipt，由 manager 统一向用户求证。

### 7.2 改造：启动参数

```typescript
/**
 * Worker 启动参数（v2）
 */
export interface WorkerRunOptions {
  /** 授权载体：当前 todo ID */
  todoId: string
  /** todo 正文快照（由 manager 折叠必要上下文） */
  todoSnapshot: string
  /** 层 1-5 的稳定前缀 system prompt */
  systemPrompt: string
  /** 按需注入的 memory recall 消息 */
  memoryRecall?: AgentMessage[]
  /** 模型配置 */

  apiKey: string
  /** 工作区路径 */
```

关键变化：

- **不再传入 manager 的 `messages` 历史** → 上下文隔离
- manager 把必要上下文折叠进 `todoSnapshot`
- worker 的 messages 数组仅包含：`memoryRecall`（0-1 条）+ `todoSnapshot` 作为 user message

### 7.4 改造：单 todo 生命周期

```typescript
const MAX_CONSECUTIVE_FAILURES = 2

export async function runWorkerAgent(options: WorkerRunOptions): Promise<WorkerResult> {
  // 1. 构建工具集（过滤掉 todo_write 和 sessions_spawn）
  const tools = createWorkerTools()

  // 2. 构建 messages（上下文隔离）
  const messages: AgentMessage[] = [
    ...(options.memoryRecall ?? []),
    { role: 'user', content: options.todoSnapshot },
  ]

  // 3. 运行 agentLoop
  //    - 工具调用受权限三档约束
  //    - confirm 档不直接询问用户，而是中止并返回 blocked

  // 4. 连续失败处理
  //    - consecutiveFailures >= MAX_CONSECUTIVE_FAILURES
  //    → receipt = { result: 'blocked', validation: '连续失败 N 次', nextHint: '建议拆分任务' }

  // 5. 正常完成
  //    → receipt = { result: lastAssistantMessage, validation: '...', nextHint: '...' }

  // 6. confirm 档操作
  //    → receipt = { result: 'blocked', validation: '需要用户确认: ...', nextHint: '回交 manager' }
}
```

## 11. 实现注意事项

1. `classifyToolPermission` 中的 bash 关键词检测不需要做到完美，false positive（误判为 confirm）可接受，false negative（漏判高风险操作）不可接受。
2. worker 上下文隔离是本包最关键的纪律：worker 不应该也不需要看到 manager 和用户之间的对话历史。
3. 权限三档的前端交互（confirm 弹窗）可以先用 WS 事件占位，前端实现留给后续前端开发包。

## 事实 4：extractSessionText 在 tool_use / tool_result parts 上的行为

**位置：** `backend/node_modules/@lecquy/shared/src/session.ts:366-383`

**实现：**
```ts
export function extractSessionText(content: unknown): string {
  if (typeof content === 'string') return content
  if (!Array.isArray(content)) {
    if (isObject(content) && typeof content.text === 'string') return content.text
    return ''
  }

  return content
    .map((part) => {
      if (typeof part === 'string') return part
      if (!isObject(part)) return ''
      if (part.type === 'text' && typeof part.text === 'string') return part.text
      if (part.type === 'file' || part.type === 'image' || part.type === 'thinking' || part.type === 'toolCall') return ''
      return typeof part.text === 'string' ? part.text : ''  // 第 379 行：tool_result 提取 text
    })
    .filter(Boolean)
    .join('\n')
}
```

**结论：**
对于 toolCall（工具调用），直接返回空字符串（第 378 行）；对于 tool_result，尝试提取 `part.text` 如果存在，否则返回空（第 379 行）。序列化历史时工具调用不贡献文本，但工具结果的纯文本部分被保留。

## 事实 5：compact.test.ts 当前的 mock 方式

**位置：** `backend/src/memory/compact.test.ts:14-20`

**实现：**
```ts
function createManager(workspaceDir: string): SessionManager {
  return new SessionManager({
    cwd: workspaceDir,
    sessionDir: '/tmp',
    persist: false,
  })
}
```

**使用场景（第 26 行）：**
```ts
const manager = createManager(workspaceDir)
// ... 追加消息
assert.equal(await applyCompactionIfNeeded(manager), true)
```

**结论：**
采用构造函数注入方式，直接 new SessionManager 实例并通过参数配置。不使用模块级 mock 或依赖注入容器，persist 参数设为 false 避免磁盘 I/O。这个方式支持 Phase 1 的单测需求，LLM 调用部分需要后续 mock 补充（当前测试未涉及）。

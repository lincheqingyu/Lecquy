# Prompt 序列化器 技术规范

更新日期：2026-04-10

所属开发包：包 1（Prompt Serializer 基座）
前置依赖：包 0（类型基座）
预估工期：2-3 天

## 1. 目标

新建唯一序列化入口 `prompt-serializer.ts`，让 `role: system` 字段输出 `<LAYER:xxx>` 分层格式。同时改造现有 `system-prompts.ts` 和 `prompt-module-files.ts`，使各章节返回 `LayerSlice` 而非自行拼接字符串。

旧 `buildSystemPrompt` 保留为 `buildSystemPromptLegacy` 做 fallback，新代码统一走 `buildLayeredSystemPrompt`。

## 2. 文件变更清单

|---|---|---|
| `core/prompts/prompt-serializer.ts` | **新建** | 唯一序列化入口 |

| `core/prompts/prompt-module-files.ts` | 改造 | 模板渲染产物归入对应 `LayerSlice` |

## 3. 新建 `prompt-serializer.ts`

### 3.1 核心序列化函数

```typescript
import { LayerSlice, PromptLayer, LAYER_TAGS, PREFIX_CACHE_LAYERS } from './prompt-layer-types'
import { createHash } from 'crypto'

/**
 * 纯函数：将多个 LayerSlice 序列化为单条 systemPrompt 字符串
 * 同一输入 → 同一字节输出（确定性）
 *
 * 规则：
 * 1. 按 layer 枚举值升序排列
 * 2. 每层包裹 <LAYER:tag>\n...\n</LAYER>
 * 3. 层间固定 \n\n 分隔
 * 4. 空内容的层跳过（不输出空 LAYER 标签）
 * 5. 仅接受 prefix cache 段的层（1-5），拒绝 6-7
 */
export function serializeSystemPrompt(slices: LayerSlice[]): string
```

实现要点：

- 排序使用 `slice.layer` 数值，不依赖数组传入顺序
- 只处理 `PREFIX_CACHE_LAYERS` 范围内的层，传入 MemoryRecall/LiveTurn 时抛错
- 层间分隔固定为 `\n\n`（两个换行符），不多不少
- 空 content 的 slice 直接跳过，不生成空 `<LAYER:xxx></LAYER>`

### 3.2 mode 标签属性

`<LAYER:mode>` 需要携带 `name` 属性：

```
<LAYER:mode name="simple">
...
</LAYER>
```

`<LAYER:skill>` 需要携带 `id` 属性：

```
<LAYER:skill id="pdf">
...
</LAYER>
```

其他层不携带属性。序列化函数需支持可选的 `attributes` 字段：

```typescript
export interface LayerSlice {
  // ... 已有字段
  /** 可选属性，会渲染为 <LAYER:tag key="value"> */
  attributes?: Record<string, string>
}
```

### 3.3 辅助函数

```typescript
/**
 * 计算内容的 SHA256 哈希
 */
export function hashContent(content: string): string

/**
 * 粗估 token 数（1 token ≈ 3.5 字符）
 */
export function estimateTokens(content: string): number

/**
 * 构建单个 LayerSlice
 */
export function createSlice(
  layer: PromptLayer,

```

### 3.4 顶层编排函数

```typescript
/**
 * 替代原 buildSystemPrompt，统一走分层序列化
 */
export function buildLayeredSystemPrompt(
  options: BuildLayeredPromptOptions
): LayeredPromptResult {
  // 1. 构建各层 slice
  const systemSlice = buildSystemLayerSlice(options)
  const modeSlice = buildModeLayerSlice(options)
  const startupSlice = buildStartupLayerSlice(options)
  const skillSlice = buildSkillLayerSlice(options)
  const prefSlice = buildUserPreferenceSlice(options)

  // 2. 序列化
  const allSlices = [systemSlice, modeSlice, startupSlice, skillSlice, prefSlice]
  const systemPrompt = serializeSystemPrompt(allSlices)

  // 3. 收集哈希和 token 信息
  return { systemPrompt, sliceHashes, sliceTokens }
}
```

### 4.2 `buildProjectContextSection` 拆分

当前该函数加载所有 context files（SOUL/IDENTITY/AGENTS/USER/TOOLS/MEMORY）并拼成一个大字符串。需要拆分为：

| 内容 | 归属层 | 新的数据来源 |
|---|---|---|
| SOUL.md | startup context (层 3) | 由包 2 的 `loadStartupSlices` 提供 |
| IDENTITY.md | startup context (层 3) | 由包 2 的 `loadStartupSlices` 提供 |
| USER.md profile | startup context (层 3) | 由包 2 的 `loadStartupSlices` 提供 |
| USER.md preference | user preference (层 5) | 由包 2 的 `loadStartupSlices` 提供 |
| MEMORY.md | **不再走此路径** | 由包 3 的 memory 路径提供 |
| AGENTS.md (动态生成) | system (层 1) | `buildManagedAgentsContent()` 归入 system 层 |
| TOOLS.md (动态生成) | system (层 1) | `buildManagedToolsContent()` 归入 system 层 |

### 4.3 各层 slice 构建函数

```typescript
/**
 * 构建 system 层 slice
 * 合并：identity + tooling + tool-call-style + safety + skills-index
 *       + workspace + documentation + time + runtime + extra-instructions
 *       + AGENTS.md 内容 + TOOLS.md 内容
 */
export function buildSystemLayerSlice(options: BuildLayeredPromptOptions): LayerSlice

/**
 * 构建 mode 层 slice
 * 内容：role directive（simple/manager/worker 差异部分）
 *       + 通用「结果/验证/下一步」答复结构
 */
export function buildModeLayerSlice(options: BuildLayeredPromptOptions): LayerSlice

/**
 * 构建 startup context 层 slice
 * 内容：CAPABILITY block + SOUL + IDENTITY + USER.profile + MEMORY.summary
 * 包 1 阶段：直接使用 options 中传入的内容
 * 包 2 阶段：由 loadStartupSlices 提供
 */
export function buildStartupLayerSlice(options: BuildLayeredPromptOptions): LayerSlice

/**
 * 构建 skill runtime 层 slice
 * 包 1 阶段：如果有 activeSkillName 则加载 skill 正文，否则返回空
 * 包 5 阶段：由 SkillSession 管理
 */
export function buildSkillLayerSlice(options: BuildLayeredPromptOptions): LayerSlice

/**
 * 构建 user preference 层 slice
 * 内容：USER.md preference 切片
 */
export function buildUserPreferenceSlice(options: BuildLayeredPromptOptions): LayerSlice
```

## 5. 改造 `prompt-module-files.ts`

### 5.1 当前状态

14 个模板通过 handlebars 风格变量替换渲染，产物自行拼接到大字符串中。

### 5.2 改造方式

模板渲染机制保持不变（handlebars 替换仍然保留），但渲染产物不再自行拼接。每个模板的渲染结果作为字符串返回给调用方（`system-prompts.ts` 中的 buildXxxSection 函数），由调用方归入对应 `LayerSlice`。

具体改动：

- `renderPromptModule(name, vars)` 签名不变，仍返回 `string`
- 调用方（各 `buildXxxSection`）负责把返回值归入正确的 `LayerSlice`
- `.lecquy/system-prompt/` 的覆盖机制保持不变

## 7. 验收标准

### 7.1 序列化稳定性测试

| # | 用例 | 预期 |
|---|---|---|
| 1 | 同一 `LayerSlice[]` 调用 `serializeSystemPrompt` 100 次 | 输出字节完全一致 |
| 2 | `startup` 层内容变化 | `system` 和 `mode` 段的字节不变 |
| 3 | `skill` 层从空变为有内容 | `system`、`mode`、`startup` 段字节不变 |
| 4 | `user_preference` 层变化 | 前 4 层字节不变 |
| 5 | 传入 `PromptLayer.MemoryRecall` 的 slice | 抛出错误 |

### 7.2 协议测试

| # | 用例 | 预期 |
|---|---|---|
| 1 | 输出字符串中搜索 `<LAYER:memory_recall>` | 不存在 |
| 2 | 输出字符串中搜索 `<LAYER:live_turn>` | 不存在 |
| 3 | `<LAYER:mode>` 标签包含 `name` 属性 | `<LAYER:mode name="simple">` 或 `<LAYER:mode name="plan">` |
| 4 | 空 content 的 slice | 对应 LAYER 标签不出现在输出中 |
| 5 | 层间分隔 | 恰好 `\n\n`，不多不少 |

### 7.3 回归测试

| # | 用例 | 预期 |
|---|---|---|
| 1 | `buildLayeredSystemPrompt` 输出与 `buildSystemPromptLegacy` 输出做 diff | 所有原有信息在新输出中均存在（位置可不同） |
| 2 | `LAYERED_PROMPT=true` 启动后端，跑 simple 模式端到端 | 正常对话、工具调用通过 |
| 3 | `LAYERED_PROMPT=false` 启动后端 | 旧路径正常工作，无回归 |

## 9. 关键实现约束

1. `serializeSystemPrompt` 必须是纯函数，不读文件系统、不读环境变量、不产生副作用。
2. 标签名硬编码在 `LAYER_TAGS` 常量中，不从配置文件读取。
3. 层间分隔的换行数量（2 个 `\n`）硬编码，不可配置。
4. `buildLayeredSystemPrompt` 中调用 `buildStartupLayerSlice` 时，包 1 阶段直接使用 `options` 传入的内容字符串；包 2 完成后切到 `loadStartupSlices` 产物。
